GitHub Basics

GitHub is a platform for hosting and collaborating on code repositories using Git. It allows developers to manage version control, collaborate with others, and deploy code seamlessly.

Steps to Initialize, Add, Commit, and Push Code

  1. Initialize a Git repository:
    Use git init to initialize a new Git repository in your project folder.
    git init
  2. Add files to staging:
    Use git add to stage files for a commit. To stage all files:
    git add .
  3. Commit your changes:
    Use git commit to save your changes with a message describing the changes.
    git commit -m "Your commit message here"
  4. Push changes to a remote repository:
    Add a remote repository (if not already added) using:
    git remote add origin https://github.com/username/repository.git
    Push your changes to the remote repository using:
    git push -u origin main

Example Workflow

git init
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/username/my-project.git
git push -u origin main
